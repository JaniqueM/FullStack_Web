## Setup Instructions

### Step 1 вЂ” Clone the repository
```bash
git clone <your-repo-url>
cd sms_api_project
```

### Step 2 вЂ” Create and activate a virtual environment
```bash
python -m venv venv

# On Windows:
venv\Scripts\activate

# On Mac/Linux:
source venv/bin/activate
```

### Step 3 вЂ” Parse the XML data
Navigate to the `dsa/` folder and run the parser:
```bash
cd dsa
python parse_sms.py
```
This will generate a `transactions.json` file with all 1691 SMS records.

### Step 4 вЂ” Copy transactions.json to the api/ folder
```bash
copy transactions.json ../api/transactions.json   # Windows
cp transactions.json ../api/transactions.json      # Mac/Linux
```

### Step 5 вЂ” Start the API server
Navigate to the `api/` folder and run:
```bash
cd ../api
python api.py
```

You should see:

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/transactions` | Get all transactions |
| GET | `/transactions/{id}` | Get one transaction by ID |
| POST | `/transactions` | Add a new transaction |
| PUT | `/transactions/{id}` | Update an existing transaction |
| DELETE | `/transactions/{id}` | Delete a transaction |

All endpoints require **Basic Authentication**:
- Username: `admin`
- Password: `password123`

See `docs/api_docs.md` for full documentation with request and response examples.

---

## Running the DSA Comparison

Navigate to the `dsa/` folder and run:
```bash
cd dsa
python dsa_search.py
```

This compares the speed of **Linear Search** vs **Dictionary Lookup** across 20 records and prints a results table.

---

## Authentication

This API uses Basic Authentication. Note that Basic Auth is not recommended for production use as credentials are only Base64 encoded, not encrypted.

Stronger alternatives include:
- **JWT (JSON Web Tokens)**
- **OAuth2**
- **API Keys**
